# Graduation Project — Система управления сервисными работами

Курсовой/дипломный проект Ahmed Kashima: полнофункциональная система для управления сервисными заявками на обслуживание генераторов. Проект объединяет веб‑интерфейс (админ и сотрудники) и API на Flask с базой данных, аналитикой и встроенным чатом.

**Кратко**
- Назначение: управление клиентами, генераторами и сервисными заданиями
- Роли: администратор и техник (пользователь)
- Ключевые модули: расписание работ, админ‑панель, чаты, аналитика

**Основные возможности**
- Аутентификация сотрудников по JWT и разграничение прав
- Управление клиентами и их контактными данными
- Каталог генераторов и импорт из CSV
- Создание, редактирование и закрытие сервисных задач
- Назначение техников на задачи
- Фильтрация расписания по диапазону дат
- Админ‑фильтры по активным проектам, статусам и периодам
- Реальный‑временной чат между сотрудниками
- KPI‑карточки и графики для админ‑аналитики

**Роли и права**
| Роль | Доступ |
| --- | --- |
| Администратор | Полное управление сотрудниками, клиентами, задачами, генераторами, аналитикой и чатами |
| Пользователь (техник) | Просмотр назначенных задач, обновление статуса, общение с админом |

**Ключевые сценарии**
- Админ создает клиента и задачу, привязывает генератор и тип услуги
- Админ назначает техников на задачу, редактирует и закрывает задачу
- Пользователь видит только свои задачи и отмечает выполнение
- Админ просматривает активные проекты и аналитику по выручке/активности
- Команда общается через встроенный чат

**Интерфейс и страницы**
- `Главная` — быстрый доступ к разделам, статус роли
- `Расписание` — список задач с фильтром по датам и карточками работ
- `Клиенты` — создание клиентов, поиск и просмотр карточек
- `История клиента` — профиль клиента, создание задач, история работ
- `Команда` — управление сотрудниками и их правами
- `Админ‑панель` — KPI, активные проекты и аналитика
- `Чат` — прямые диалоги сотрудников и администраторов

**Задачи и генерация работ**
- Создание задачи происходит в профиле клиента
- Для задачи выбираются дата, время, генератор, тип услуги и заметки
- Задача попадает в расписание и может назначаться на техников

**Фильтры и поиск**
- Расписание фильтруется по диапазону дат
- Админ‑клиенты фильтруются по активности и статусу проектов
- Админ‑проекты фильтруются по городу, типу услуги и технику
- Поиск клиентов по имени/фамилии/контактам

**Аналитика и графики**
- KPI‑карточки: клиенты, активные проекты, сообщения за день, выручка
- График активности по дням
- Месячная выручка по завершенным работам
- Дополнительные виджеты: статусы задач (круговая), производительность техников (столбиковая)

**Чат и уведомления**
- Комнаты вида `user_<id>` для приватных диалогов
- Поддержка индикатора набора и статуса прочтения
- Список диалогов с непрочитанными сообщениями

**Технологии**
- Frontend: React, TypeScript, Vite, Tailwind CSS, Recharts
- Backend: Flask, SQLAlchemy, Flask‑JWT‑Extended, Flask‑SocketIO
- База данных: SQLite + Alembic миграции

**Структура проекта**
- `src/` — фронтенд (страницы, компоненты, маршруты)
- `backend/` — API, модели БД, миграции, Socket.IO
- `public/` — статические ресурсы

**API (кратко)**
- Аутентификация: `POST /token`, `POST /logout`, `GET /profile`
- Сотрудники: `GET /employees`, `POST /employees/create`, `POST /employees/delete`, `POST /employees/permission`
- Клиенты: `POST /customer/display`, `POST /customer/details`, `POST /customer/create`, `POST /customer/delete`
- Генераторы: `GET /generators/details`, `GET /generator/create`
- Задачи: `POST /service/create`, `POST /service/details`, `POST /schedule/*`
- Аналитика: `GET /api/admin/metrics`, `GET /api/admin/activity`, `GET /api/dashboard/*`
- Чаты: `GET /api/conversations/<user_id>`, `POST /api/conversations`, Socket.IO события

**Запуск проекта**
Frontend:
```bash
npm install
npm run dev
```

Backend (пример):
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r backend/requirements.txt
export SECRET_KEY="your-secret"
python backend/base.py
```

Примечание: фронтенд ожидает API на `http://127.0.0.1:3000`.
